✅ LeetCode — Merge Sorted Array (LC 88)

🔹 Problem Statement (Simplified)
You are given two sorted arrays nums1 and nums2.
nums1 has extra space at the end to hold nums2.
Merge nums2 into nums1 so the final array remains sorted.
Do it in-place without returning a new array.

🔹 Example

Input:
nums1 = [1,2,3,0,0,0], m = 3
nums2 = [2,5,6],       n = 3
Output:
[1,2,2,3,5,6]


🔹 Interview-Preferred Approach: 3 Pointers (from the end)
Why merge from end?
Avoids shifting elements
In-place → no extra array
Optimal and clean

🔹 Algorithm
i = m-1 (last real element in nums1)
j = n-1 (last element in nums2)
k = m+n-1 (last index of nums1)
Compare nums1[i] and nums2[j], put the larger at k
Move pointers
If nums2 still has elements, copy them


class Solution {
    public void merge(int[] nums1, int m, int[] nums2, int n) {
        int i = m - 1;
        int j = n - 1;
        int k = m + n - 1;

        while (j >= 0) {
            if (i >= 0 && nums1[i] > nums2[j]) {
                nums1[k--] = nums1[i--];
            } else {
                nums1[k--] = nums2[j--];
            }
        }
    }
}


⏱ Time & Space Complexity
Time: O(m + n)
Space: O(1) auxiliary (in-place merge)

🔥 Interview Tip
If asked:
Why not merge from the start?
Answer:
It requires shifting elements. Merging from the end avoids shifts and keeps it optimal.

📌 Quick Revision Table
Approach	Time	Space	In-place
Merge from start	O(m+n)	O(1)	❌ Needs shifts
Optimized (from end)	O(m+n)	O(1)	✅ No shifts
