## 📚 Topics Covered

### 0.1 What is Time Complexity?
Understanding how algorithm performance scales with input size.

### 0.2 Asymptotic Notations
- Big-O (Upper bound)
- Big-Ω (Lower bound)
- Big-Θ (Tight bound)

### 0.3 Best, Average, Worst Case
Analyzing algorithm behavior in different input conditions.

### 0.4 Common Time Complexities
- `O(1)` – Constant
- `O(log n)` – Logarithmic
- `O(n)` – Linear
- `O(n log n)` – Linearithmic
- `O(n²)` – Quadratic
- `O(2ⁿ)` – Exponential

### 0.5 Why Binary Search is `O(log n)`
Because it divides the input space in half at every step.

### 0.6 Space Complexity
Covers:
1. Input space
2. Auxiliary space
3. Recursion stack space

### 0.7 Amortized Analysis
Analyzing average cost over multiple operations (e.g., dynamic array resizing → amortized `O(1)`).

### 0.8 Time vs Space Trade-off
- More memory → faster execution (Hashing)
- Less memory → slower execution (Sorting/Loops)

## 🎯 Placement-Focused Mindset

- Think before you code

- Analyze constraints
- Optimize for **1-pass** when possible
- Evaluate recursion stack impact
- Avoid brute force unless constraints are very small
