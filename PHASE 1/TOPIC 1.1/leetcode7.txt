7️⃣ Reverse Integer (LeetCode 7)

🔹 Problem Statement (Simplified)
Given a 32-bit signed integer x, return its digits reversed.
If reversed value goes outside 32-bit range, return 0.

🔹 Example

Input: 123
Output: 321

Input: -123
Output: -321

Input: 120
Output: 21

🔹 Interview-Preferred Approach: Math + Overflow Safe
Why this?

No string conversion

O(1) memory

Handles overflow safely

🔹 Algorithm
1.Initialize rev = 0
2.Loop while x ≠ 0
3.Extract last digit → digit = x % 10
4.Check overflow before updating rev
5.Update → rev = rev * 10 + digit
6.Remove digit → x /= 10
7.Return rev or 0 if overflow

🔹 Java Code (BEST ANSWER)

class Solution {
    public int reverse(int x) {
        int rev = 0;
        while (x != 0) {
            int digit = x % 10;

            // overflow check
            if (rev > Integer.MAX_VALUE / 10 || rev < Integer.MIN_VALUE / 10)
                return 0;

            rev = rev * 10 + digit;
            x /= 10;
        }
        return rev;
    }
}

🔹 Time & Space Complexity

Time: O(d) → d = number of digits

Space: O(1)

🔥 Interview Tip
If interviewer asks:

Why check overflow before multiplication?

Answer:

Because rev * 10 can exceed 32-bit limit, so we validate first to avoid error.
